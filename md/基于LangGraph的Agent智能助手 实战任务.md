## 三、分阶段实战任务
### 阶段1：项目解读与环境准备（1h）
1. **文档精读**
    - 梳理系统架构：3个Agent分工+工作流链路
    - 明确状态定义：`TravelInfo`字段作用
    - 掌握LangGraph核心：`StateGraph`、节点、边、状态传递
2. **环境搭建**
    - 创建项目目录结构（严格匹配讲义）
    - 安装依赖：`langgraph`、`langchain_deepseek`、`pydantic`、`asyncio`
    - 配置DeepSeek API Key

### 阶段2：核心模块编码实现（2h）
按讲义逐文件实现，**禁止直接复制，手动敲写并理解逻辑**：
1. **状态定义（states.py）**
    - 实现`PlanningState`枚举
    - 用Pydantic定义`TravelInfo`数据模型
2. **Agent模块（agents/目录）**
    - 天气Agent：固定JSON输出、异常处理、异步调用
    - 景点Agent：多维度推荐、温度参数设置
    - 行程Agent：分日行程、交通餐饮建议整合
3. **编码要求**
    - 注释关键代码
    - 保留异常处理逻辑
    - 严格遵循JSON输出格式

### 阶段3：工作流整合与调试（2h）
1. **主程序实现（main.py）**
    - 初始化3个Agent
    - 用`StateGraph`搭建工作流：添加节点→设置入口→连接边→编译
    - 编写异步`main`函数，设置初始旅行参数
2. **调试运行**
    - 解决API调用、JSON解析、状态传递报错
    - 运行成功输出完整旅行计划
    - 替换目的地（如北京、上海）验证通用性

### 阶段4：功能优化与拓展（1.5h）
完成**必做优化**+**二选一拓展**：
#### 必做优化
1. 为每个Agent添加日志输出
2. 优化异常返回值，更贴合真实场景
3. 统一输出格式，提升可读性
#### 拓展任务（二选一）
1. **条件分支**：根据天气（雨天/晴天）跳转不同推荐逻辑
2. **新增功能**：添加预算计算Agent，生成每日花费预估

### 阶段5：成果验收与总结（0.5h）
1. **验收标准**
    - 代码可独立运行，无报错
    - 输出完整天气、景点、行程信息
    - 完成至少1项优化/拓展
2. **总结输出**
    - 1分钟口述：项目流程+核心知识点
    - 提交完整代码+运行截图
